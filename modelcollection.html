Model CollectionÂ¶

To work with a framework for hydrological modelling is all the more useful, the more models are implemented. However, HydPy is not only intended to allow for the flexible combination of different model concepts. It tries to contribute to the improvement of software quality and transparency in hydrological modelling. Hence, each model shall be implemented in accordance with the HydPy-Styleguide. We decided to publish only those of our own model implementations, which fullfill this requirement.

Models presently available:

  • HydPy-A
    • arma
    • arma_v1
  • HydPy-D
    • dam
    • dam_v1
  • HydPy-H (HBV)
    • HydPy-H-Land
    • HydPy-H-Stream
    • HydPy-H-Branch
    • hland_v1 (HBV96 version of HydPy-H-Land)
    • hstream_v1 (HBV96 version of HydPy-H-Stream)
    • hbranch_v1 (HBV96 version of HydPy-H-Branch)
  • HydPy-L (LARSIM)
    • HydPy-L-Land
    • HydPy-L-Stream
    • HydPy-L-Lake
    • lland_v1 (LARSIM-Xinanjiang-Turc-Wendling version of HydPy-H-Land)
    • lland_v2 (External potential evaporation version of lland_v1)
    • lstream_v1 (LARSIM-Manning version of HydPy-H-Stream)
    • llake_v1 (LARSIM-Lake version of HydPy-H-Branch)
